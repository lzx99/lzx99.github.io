GIT问题

git可以从远程checkout，不小心把重要分支合乱以后。

git每一次切换分支文件都是全局切换，所有不同分支的不同文件不会untracked（未暂存文件）。

新建文件在每一个分支都是untracked，注意git merge后，会将被merge分支的没有被merge的文件在merge的分支变成未暂存文件（会检查小分支上有哪些文件新建和修改，新文件直接merge进分支，如果是修改的会提示解决冲突），merge后通常需要配合commit。

git reset 可以将整个版本退步。

git的核心是在小分支上新增和修改，往大分支上合。

文件修改就变成modified（暂存文件），存在暂存文件需要切换分支时，要提交或者git stash后保存暂存区状态，才能checkout。idea提供smart checkout（保留暂存区切换到另一个分支）和force checkout，看情况使用。

git merge如果有冲突文件，idea/source tree都能选择保留哪个分支的更改。
