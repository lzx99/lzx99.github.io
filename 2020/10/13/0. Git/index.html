Git还原

场景1：从暂存区还原：git checkout -- a.txt。

场景2：git reset HEAD <file>（将最新分支恢复到暂存区）

场景3：历史版本直接回退到工作区和暂存区

git reset --hard commit_id 以hard模式回退版本，HEAD指向的版本为当前版本，所以用HEAD^。

Git删除

需要用命令git rm用于删除一个文件，操作保存在暂存区，git commit -m提交版本

clone

不指定分支clone时，默认下载远程项目（master分支）， 会关联并追踪所有分支，默认名为origin

分支：可以git checkout -b dev origin/dev在本地创建。

remote

创建本地库后，本地库与自己的远程库关联并设定主机名（默认远程库为origin），一般用于本地库上传到GitHub的空库（remote以后push一个master分支）

  • 查看本地关联的远程库信 息：git remote -v

Push

推送本地分支到远程，没有则创建，如果分支在远程分支的后面，解决冲突才能推送，-f参数强制覆盖远程的，一般用不到，一般只需要同步master分支和dev分支，feature分支取决于你是否需要修复，bug分支般只由你一个人修复，不需要同步。

  • 推送本地master分支到远程master分支：git push origin 本地master:远程master（gp）

    省略:远程分支名时，代表推送到远程同名：git push origin master

    推送同时，将远程master和本地关联：git push -u origin master（第一次）

  • 将当前推送到有origin上他的追踪关系的分支，可以省略分支名字：git push origin

  • 当只有一个主机的追踪分支时可以省略主机名：git push

  • 推送所有分支到远程：git push –all origin

  • 删除远程分支：git push origin --delete dev或者git push origin :dev

Pull = Fetch+Merge

取回并合并：git pull origin 远程master:本地master，本地没有则会创建，也可以使用

省略:本地分支时，代表取回并合并到本地当前所在分支：git pull origin master

当前分支与origin上他的追踪分支取回并合并：git pull origin

当前分支与唯一主机上的追踪分支取回并合并：git pull

设置本地dev和远程dev分支关联：git branch --set-upstream-to=origin/dev dev（或者第一次push的时候加如-u）

Fetch

用于抓取远程分支到本地，成为origin/master分支，不会合并到本地master

所以remote后，需要用git fetch origin取回所有分支信息，才能查看远程分支信息：git branch -r

分支查看：

  • 查看本地分支：git branch(gb)，当前分支会显示*
  • 查看远程分支：git branch -r(gbr)
  • 查看本地分支和远程git的分支：git branch -a(gba)
  • 创建分支：git branch dev
  • 创建并切换分支：git checkout -b dev
  • 普通删除：git branch -d <name>（gbd）
  • 删除没有被合并过的分支，git branch -D <name>（gbD）
  • 本地-远程的追踪分支关系：git branch -vv

切换分支时：git checkout/switch dev(gco dev)，切换就是HEAD指针的变化。

  • 快速合并dev分支到当前分支：git merge dev(gm)，快速合并有时会产生冲突，比如当dev提交后，master又有新的提交。此时修改冲突文件，再次add+commit，此时冲突解决，两个分支合成新的分支。
  • 禁用快速合并在（普通模式）：git merge —-no-ff lee -m "Do Job"，能看出来谁曾经做过合并。
